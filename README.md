

Это небольшой проект на Symfony, созданный в личных целях.
Цель: наведение порядка в электронных книгах.
Консольная утилита анализирует файлы с электронными книгами в указанной папке, собирает метаданные (автор, название, издание, год выпуска) и переименовывает файлы в соответствии с выбранным форматом названия.

## Требования ##

1. PHP >7.3
2. Composer


## Текущие ограничения ##

На данный момент поддерживается только формат fb2.
В будущем, планируется как минимум поддержка epub и mobi.
Есть планы на обработку pdf и djvu - файлов, даже без OCR.

Также сейчас поддерживается только один формат наименования:

```
[%язык_если_не_русский%] %Автор% - %Название% (%издание_если_есть%) [%год%].%расширение%
```

Также в дальнейшем планируется возможность вести каталог в БД, сверяться с ним.


## Установка и запуск ##

1. Склонировать проект.
2. Установить зависимости: `composer install -o`
3. Запустить из папки: `./bin/console books:process`


## Аргументы ##

 `--path (-p)` : указывает папку, в которой лежат файлы. Обязательный.  
 `--dry-run (-d)`: только сбор данных, без переименования  
 `--manual (-m)`:  ручной режим: для каждого файла предлагает вариант переименования и спрашивает  
 `--verbose (-v)`: описывает все свои действия  
 `--very-verbose (-vv)`: описывает все свои действия + выводит всю инфу, которую удалось спарсить из книги (NOT IMPLEMENTED YET)  

## Примеры использования ##


	cd path/to/project

	# Проанализировать fb2-файлы и переименовать (автоматический режим)
	./bin/console books:process -p path/to/books

	# Проанализировать fb2-файлы и переименовать (ручной режим)
	./bin/console books:process -p path/to/books -m -v

	# Проанализировать fb2-файлы и вывести предлагаемые названия
	./bin/console books:process -p path/to/books -v -d

	# Проанализировать fb2-файлы и вывести все, что удалось спарсить
	./bin/console books:process -p path/to/books -vv -d
